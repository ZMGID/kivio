# 聊天消息轨道导航

## Goal

为长对话提供一种以对话轮次为坐标的语义导航控件，使用户能够快速判断当前阅读位置、预览历史轮次并跳转，而不再只依赖按像素比例工作的传统滚动条。

该控件面向 Kivio 的复杂聊天结构，包括普通单模型回答、多模型回答组、流式回答和上下文压缩记录。

## Background and Confirmed Facts

- 当前消息列表由 `src/chat/MessageList.tsx` 统一构建并使用 `virtua` 的 `Virtualizer` 渲染。
- `VirtualizerHandle.scrollToIndex()` 已可按虚拟列表项定位，现有列表也已有滚动监听、贴底状态和“回到底部”行为。
- 同一 `group_id` 的连续 assistant 消息会在 `src/chat/messageGroups.ts` 中折叠为一个 `MessageGroup`，因此底层消息数不等于用户感知的对话轮次数。
- 列表还包含 spacer、流式占位、错误提示、压缩分界线和压缩摘要等非消息项，不能直接把全部 `RenderItem` 机械映射为轨道刻度。
- 流式内容更新频率高；导航数据应与逐帧文本更新隔离，避免整个导航轨道持续重渲。

## Requirements

- R1. 提供独立于传统像素滚动条的“消息轨道 / 对话导航条”，以稳定的语义单元表示长对话结构。
- R2. 以“一轮对话”作为一级导航单元：用户消息为轮次起点，其后的单模型回答或同组多模型回答归入该轮；每轮只显示一根一级刻度。
- R3. 多模型回答组只占一个一级导航节点，不按 assistant 消息数量拆成多根刻度。
- R4. 当前视口内可见的所有对话轮次，其普通刻度均使用深色；视口外节点保持浅色。默认静止状态不改变普通刻度形状。鼠标进入轨道后，刻度长度按其与指针的纵向距离连续变化，最近的最长、邻近的逐级缩短。节点较少时整组轨道垂直居中，刻度与间距保持紧凑。
- R5. 用户悬停导航节点时显示预览卡：用户问题最多 2 行、assistant 回答摘要最多 3 行，底部显示单模型名称或多模型数量等聚合信息。
- R6. 用户点击导航节点后，列表应定位到该轮的稳定起点，并尊重系统“减少动态效果”设置。
- R7. 当前轮次判定应具有稳定的阅读基准，避免因消息高度变化、Markdown 延迟布局或轻微滚动而频繁跳动。
- R8. 导航结构不得订阅每一帧流式文本；生成过程中只更新必要的当前状态。
- R9. 每个对话轮次始终保留独立刻度，不合并、不聚类；节点超出轨道可用高度时，轨道本身提供纵向滚动以访问全部轮次。
- R10. 不破坏现有的虚拟列表、流式钉底、向上滚动解除跟随、“回到底部”、多模型横向滚动和上下文压缩展示。
- R11. 轨道常驻在消息正文左侧，仅在对话达到显示条件时出现；窄窗口自动隐藏，中等宽度必须为轨道保留独立安全区，不得与 Markdown 正文重叠。
- R12. MVP 阶段保留并继续显示右侧传统滚动条；左侧消息轨道作为新增的语义导航与其并存。
- R13. 鼠标位于轨道上滚轮时，按对话轮次导航并同步滚动正文；正文滚动时轨道自动跟随并保持当前轮节点可见，轨道不得停留在与正文当前位置脱节的状态。
- R14. 已完成的上下文压缩记录作为可跳转的特殊节点显示，使用不同于普通轮次的视觉形状；悬停显示压缩说明和摘要片段，点击定位到对应压缩摘要。
- R15. 轨道仅在至少存在 4 轮对话且消息区域宽度不小于 720px 时显示；特殊压缩节点不计入轮次数量。

## Acceptance Criteria

- [ ] AC1. 一段包含普通问答与多模型回答组的测试对话，轨道节点数量和顺序与定义后的“对话轮次”一致。
- [ ] AC2. 滚动消息列表时，激活节点能够稳定跟随当前阅读轮次，长回答内部滚动不会无故切换到相邻轮次。
- [ ] AC3. 悬停任一可见导航节点会显示对应轮次预览；移出后预览关闭，且不会改变正文滚动位置。
- [ ] AC4. 点击任一导航节点会跳转到对应轮次起点；跳转后激活状态与正文位置一致。
- [ ] AC5. 多模型回答组在轨道中只表示为一个回答单元或一个轮次节点，不随模型数量增加节点。
- [ ] AC6. 流式生成期间轨道不会因 token 级内容更新持续整体重渲，现有贴底和用户向上翻阅行为保持正常。
- [ ] AC7. 对超出直接展示阈值的长对话，轨道仍能呈现当前位置并允许导航到任意轮次。
- [ ] AC8. 开启 `prefers-reduced-motion` 时，节点跳转不使用强制平滑动画。
- [ ] AC9. 相关单元测试覆盖轮次建模、多模型折叠、当前节点判定和长对话密度策略；现有 MessageList 测试继续通过。
- [ ] AC10. 少于 4 轮或消息区域窄于 720px 时轨道隐藏；达到两个条件后出现，传统滚动条在两种状态下均保持可用。
- [ ] AC11. 节点超出轨道高度时不发生合并；在轨道上滚轮能够逐轮导航正文，正文滚动也会将当前节点带回轨道可视区。
- [ ] AC12. 压缩分界以特殊节点出现，预览和点击定位均指向对应压缩摘要。
- [ ] AC13. 节点总高度小于轨道视口时整体垂直居中；节点较多时从可滚动内容起点正常排列。
- [ ] AC14. 在 720–871px 消息区域宽度内，轨道和激活刻度不与正文首字、列表符号或代码块发生视觉重叠。
- [ ] AC15. 鼠标未进入轨道时，所有普通节点长度一致；与正文视口相交的全部轮次节点为深色，视口外节点为浅色。
- [ ] AC16. 指针在轨道内纵向移动时，节点长度按距离形成连续的局部鱼眼效果；移出轨道后全部恢复默认长度。

## Out of Scope for MVP

- 不把工具调用、推理片段或每个 assistant segment 各自做成一级刻度。
- 不在第一版引入全文搜索、书签、手动重命名节点或跨会话导航。
- 悬停预览不展示附件列表、工具调用详情或消息时间。
- 不改变聊天消息的持久化结构。
- 不在 MVP 中隐藏或替代传统滚动条；后续可根据使用反馈逐步演进。

## Technical Notes

- 预计在 `MessageList` 的 `RenderItem[]` 之上派生稳定的导航模型，保存语义节点到目标虚拟索引的映射。
- 本任务属于复杂交互功能；开始实现前必须补齐 `design.md` 和 `implement.md`，并由用户审核规划产物。
