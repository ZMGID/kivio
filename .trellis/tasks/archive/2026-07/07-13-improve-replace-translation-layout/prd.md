# 替换翻译 V2：照片级擦除、精确几何与跨平台语义排版

## Goal

把替换翻译从“翻译分组同时决定擦除和绘制矩形”的旧实现，升级为 Windows 与 macOS 同期交付的单一新管线：精确保留原文位置和界面结构，完整翻译文本，并在 UI、文档、表格和照片场景中实现稳定的原文擦除与自然重绘。

## User Value

- 菜单、列表、按钮、段落和表格译文回到原文对应位置，不因多行合并整体上移、跨列或覆盖相邻元素。
- 原文按文字笔画擦除，保留橙色高亮、边框、图标、表格线和背景纹理。
- 译文保持完整，不通过摘要、截断或省略换取排版空间。
- macOS 与 Windows 使用一致的核心契约、质量标准和发布门禁。

## Confirmed Existing Behavior and Anchors

- `src-tauri/src/lens_commands.rs:1982-2059`：替换翻译主流程、区域生成和事件载荷的历史入口。
- `src-tauri/src/prompts.rs:203-255`：替换翻译提示词与结构化返回的历史契约。
- `src-tauri/src/rapidocr.rs:749-768`：OCR 行及其几何输出的历史来源。
- `src/lens/ReplaceTranslateOverlay.tsx`：历史上由同一 region 同时承担擦除、背景覆盖和译文绘制，容易把翻译上下文合并误当作渲染几何。
- 已确认回归包括：多行合并后首行位置上移、macOS 刘海遮挡提示、表格/列表错位、直角外框异常、保留区被误擦和原文残留。

## Requirements

- R1：替换翻译必须保留完整译文，不得要求模型摘要、压缩、截断或省略内容。
- R2：OCR 输出必须保留稳定 ID、阅读顺序、四点多边形、行/元素边界、基线、旋转和来源置信度。
- R3：系统必须把 `OcrLeaf`、`TranslationGroup`、`RenderSlot` 与 `EraseMask` 拆成四个独立契约；翻译可合并上下文，但不得因此合并绘制位置或擦除范围。
- R4：译文通过稳定 ID 映射；乱序、未知、重复、缺失、空结果和非法 JSON 均须局部回退，不能使整页失败。
- R5：默认几何策略必须保守。unknown 场景按精确行/元素重绘；低置信度时宁可少合并，也不得跨列、跨控件或把整页合成大段落。
- R6：段落流排版顶部锚定；菜单、列表和普通 OCR 行保持原首行位置；只有明确的单元格、短标签或标题策略才允许垂直居中。
- R7：完整译文依次通过结构内可用空间、换行、字号缩放和最终安全等比缩放适配；不得越界、覆盖无关结构或改变首行锚点。
- R8：擦除必须以文字笔画 mask 为目标，不得直接使用 OCR、翻译组或渲染槽的大矩形；需覆盖抗锯齿、描边、阴影和旋转文字。
- R9：擦除和修复必须保护高亮区、图标、头像、控件填色、边框、分隔线、表格线和局部渐变；所有修复路径均保证 mask 外逐像素不变。
- R10：UI/表格优先使用确定性快速修复；照片/复杂背景只在 mask 内使用 MI-GAN 或通过门禁的后续修复器，失败时安全回退。
- R11：场景路由至少区分 UI/菜单/列表、文档段落、表格/网格、照片文字与 unknown，并输出可复现的场景原因和置信度。
- R12：macOS 使用 Accessibility API，Windows 使用 UI Automation；两端适配为共享 `SemanticGeometryProvider` / `SemanticElement`，统一坐标、层级、角色、超时、错误和置信度语义。
- R13：原生语义 API 缺失、权限不足、超时、窗口消失或数据不可信时必须无感回退 OCR，不得阻塞替换翻译。
- R14：生产环境只保留一套 V2 管线。允许实施期间的短暂编译兼容或一次性协议迁移，但不保留运行时双管线、灰度开关或旧激进聚合回退。
- R15：照片/海报重绘采用“背景修复 + 确定性目标字形条件 + 风格编码 + 仿射融合”的候选方向，不以自由生成式扩散文字作为首版主路径；候选模型未通过体积、延迟、字体覆盖和质量门禁时，由 V2 geometry-core 系统字体路径处理该场景。

## Constraints

- Standard 档全部离线资源下载总量不超过 250 MB。
- High 档全部离线资源下载总量不超过 450 MB。
- UI/文档本地热路径不超过 1 秒，不含云端翻译耗时。
- 照片本地渲染路径不超过 3 秒，不含云端翻译耗时。
- 首版以功能和质量为优先，直接实现当前可稳定落地的完整新架构，不设置长期实验开关。
- 工作区存在用户的其他未提交修改，实施时必须逐文件限制改动范围，不覆盖或提交无关内容。

## Acceptance Criteria

- [ ] AC1：菜单、仓库列表和普通列表逐行对应原位置；翻译上下文合并不会移动第一行或把多行绘制成一个大矩形。
- [ ] AC2：普通段落顶部锚定，表格单元格不跨列，标题、按钮和图标保持独立。
- [ ] AC3：Retina、Windows DPI、显示缩放、多显示器和 macOS 刘海/安全区场景坐标正确，提示层始终位于可见工作区。
- [ ] AC4：橙色高亮及其他保留结构不被擦除；边框圆角、分隔线、表格线和控件形状保持正常。
- [ ] AC5：纯色 UI、代码徽章、表格、渐变、照片和旋转文字中无明显原文字形残影或矩形补丁。
- [ ] AC6：所有擦除/修复路径通过 mask 外逐像素一致性测试，相同输入重复运行产生确定性结果。
- [ ] AC7：译文完整，不出现漏字、重复、截断、省略或因错误 ID 映射而串行；局部失败只回退对应区域。
- [ ] AC8：macOS Accessibility 与 Windows UI Automation 可为支持的按钮、列表项、文本框和可访问网页提供真实元素边界；不可用时自动回退 OCR。
- [ ] AC9：UI、文档、表格、照片和 unknown 不互相强行误路由；低置信度统一进入保守精确行级路径。
- [ ] AC10：Canvas 使用清理后图片的自然尺寸作为 backing size，缩放后几何与 OCR/语义坐标保持一致。
- [ ] AC11：Standard/High 包体和 UI/文档、照片热路径分别满足已确认的资源与性能预算，并记录冷/热延迟和峰值内存。
- [ ] AC12：照片候选模型只有通过独立视觉、性能、体积和字体覆盖门禁才进入产品；否则自动使用 V2 系统字体重绘，不回退旧管线。
- [ ] AC13：Windows 与 macOS 均通过版本化视觉夹具的几何误差、跨区重叠、原文残影、结构保护、译文完整性和性能门禁，且生产代码只有一个行为来源。

## Child Task Map and Dependencies

1. `07-14-replace-translation-visual-benchmark`：先建立夹具、指标和基线，并贯穿所有阶段；最后负责总门禁。
2. `07-14-replace-translation-geometry-core`：首要实现依赖，完成四类契约、稳定 ID、精确 slot 和自然尺寸渲染。
3. `07-14-replace-translation-semantic-routing`：依赖 geometry-core；实现 macOS Accessibility、Windows UI Automation、文档布局评估和场景路由。
4. `07-14-replace-translation-stroke-erasure`：依赖 geometry-core；与 semantic-routing 可并行，完成笔画 mask、结构保护和修复分流。
5. `07-14-replace-translation-scene-rendering`：依赖 geometry-core、semantic-routing 与 stroke-erasure；候选模型通过门禁后才接入。
6. visual-benchmark 最后在 Windows 与 macOS 上完成联合验收，达标后父任务才可结束。
