# PRD — 恢复 RapidOCR 双档位(标准 v5-mobile / 高 v6-medium)

## 背景 / 问题
替换翻译当前唯一的离线 OCR 是 PP-OCRv6 medium(~139MB)。实测(冷启动预热命中后)`predict(det+rec)` 纯推理 **~4s**,是替换翻译总耗时(~7s)里最大的一块,且无免费调优空间(线程已满核、检测已缩到 960)。用户对比网易/有道觉得慢。

历史上曾有"标准/高"双档位,在提交 `3f51ec3`(去掉 standard 档)+ `781b4e9`(清理命名)中被移除,只剩 v6 medium。现在要把它加回来。

## 目标
恢复两档可选的离线 OCR 模型:
- **标准 = PP-OCRv5 mobile**(det 4.6MB + rec 15.8MB + dict,来自 GreatV/oar-ocr v0.3.0 release,~20MB):CPU 检测比 v6-medium 快 ~3.4×,预期把 OCR 段从 ~4s 降到 ~1–1.5s。屏幕文字(UI/文档/表格)精度足够。
- **高 = PP-OCRv6 medium**(现状不变,存 `high/` 子目录,~139MB):精度更高,留给需要时手动选。

用户可切换;每档独立按需下载、独立就绪校验。

## 范围
- 恢复 `OcrModelTier { Standard, High }` 及各处档位参数(设置、安装/状态命令、pipeline 选择、检测阈值)。
- 前端设置页恢复档位选择器 + i18n「标准 / 高」。
- 保留本次已加的:OCR 内部计时插桩、替换翻译入口的 OCR 预热(预热要针对**当前选中档**)。

## 非目标
- 不动 Apple Vision / Windows 系统 OCR 路径(另一条 macOS 加速方案,单独议)。
- 不改翻译(LLM)那一段的速度。
- 不做 inpainting / 排版相关改动。
- 不新增第三档或量化模型。

## 关键决策
1. **默认档**:
   - 截图翻译 / 替换翻译 → **默认「标准」(v5,快)**(用户明确要求,面向速度)。
   - 知识库入库(document processing)→ **默认「高」(v6,精度)**(沿用旧设计:入库不在乎慢、要识别质量)。
   - 决策点:是否两处都默认标准?当前按"各自旧默认"处理,评审时确认。
2. **迁移 / 不重复下载**:现有用户已装 v6 到 `high/`,必须继续作为「高」档有效,不被作废(沿用 `781b4e9` 保留 `high/` 子目录的做法)。默认翻转到「标准」后,v6-only 用户首次用替换翻译会提示下载那 ~20MB 的 v5——走现有"模型缺失→前端下载按钮"路径,不能崩。
3. **检测阈值分档**:v5(标准)用 oar-ocr 默认 `0.3/0.6/1.5`;v6(高)用 `0.2/0.45/1.4`。
4. **恢复方式**:以 `git show 3f51ec3^:<file>` / `781b4e9^:<file>` 为参照,在**当前 HEAD + 本次未提交的计时/预热改动**之上重新接线;**不做** `git revert`(会和计时/预热改动冲突)。

## 验收标准
- [ ] 编译通过(`cargo check --lib` + `npm run typecheck`);Rust 既有测试基线不回归。
- [ ] 设置页出现「标准 / 高」档位选择器(截图翻译 + 知识库两处),i18n 中英齐。
- [ ] 两档都能独立安装、独立显示就绪状态(`RapidOcrStatus { standard_available, high_available }`)。
- [ ] 选「标准」时:加载 v5 mobile(根目录 det/rec/keys)+ 默认检测阈值;选「高」时:加载 v6(`high/`)+ v6 阈值。
- [ ] 现有已装 v6 的用户,「高」档仍显示已就绪,不触发重下。
- [ ] 替换翻译、截图翻译、知识库图片 OCR 三条路径都按各自配置的档位走。
- [ ] 本次的 OCR 计时插桩与替换翻译入口预热保留;预热针对当前选中档。
- [ ] 实测:选「标准」后替换翻译的 `predict` 段明显下降(目标 ~1–1.5s 量级)。

## 风险
- 工作区当前有大量未提交改动(186 文件,含一个进行中的 regions→groups/slots 半成品重构,typecheck 已红)。本任务改动要与之隔离,不掺进那个半成品;typecheck 的既有红项需区分是不是本任务引入。
- v5 mobile 精度在复杂照片背景弱于 v6;默认标准后个别场景识别变差属预期,用户可切「高」。
